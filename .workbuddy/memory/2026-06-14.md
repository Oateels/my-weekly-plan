## 修复：getWeekKey 周日翻周 bug 导致数据"丢失" (v35)

- **症状**：用户刷新后本周所有数据（日记、任务、睡眠、会议等）都不见了
- **根因**：`getWeekKey()` 使用公式 `ceil((dayOfYear + jan1.getDay() + 1) / 7)`，在**每个周日**会提前翻到下一周。例如 6月14日(周日)计算为 W25，但 ISO 标准应为 W24。数据存在 `wb-2026-W24-*` 下，但页面刷新后查 `wb-2026-W25-*`，找不到数据。
- **验证**：该 bug 在 6月13日(周六)不触发(W24=W24)，6月14日(周日)触发(W25≠W24)，6月15日(周一)恢复(W25=W25)。同样在 6月21日(周日)和所有跨年周日都触发。
- **修复**：
  1. `getWeekKey()` 和 `getWeekKeyForDate()` 改用 ISO 8601 标准算法（1月4日所在周为第1周）
  2. 启动时自动对比新旧公式，如果 key 不同则自动将旧 key 下的数据迁移到新 key（数据自动找回）
