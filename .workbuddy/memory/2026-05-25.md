- **根因**：`loadFans()` 在第 923 行被调用，但 `FANS_IDS` 数组在第 935 行才定义。由于 `var` 声明提升但值仍为 `undefined`，`loadFans()` 内部迭代 `FANS_IDS.length` 时抛出异常被 try/catch 静默吞掉。
- **现象**：v14 保存正常（绿色时间戳出现，因为 v14 的 `svFans()` 用自己闭包内的 `F` 变量），但刷新后加载失败（`loadFans()` 用 `FANS_IDS`）。

- **文件**：`index.html`、`周计划模板.html` 均已修复。

## v17: 修复健身数据(exe)刷新后丢失 bug
- **根因**：`renderExeSessions()` 在构建 session DOM 之前就执行了 `exe-goal.value = goal`。`exe-goal` 上绑定了 `onchange="updateExeSessions()"`，若此操作触发 change 事件，`updateExeSessions()` → `saveExeData()` 会读到空 DOM（session 容器尚未渲染），从而将空 sessions 写入 localStorage，覆盖已保存的正确数据。
- **现象**：用户填写健身后 SAVE 日志正常，刷新后 LOAD 日志显示数据被加载，但第二次刷新数据消失（因为第一次加载时 localStorage 已被空数据覆盖）。
- **修复**：将 `document.getElementById('exe-goal').value = goal` 移到 session DOM 构建完成之后，确保即使 onchange 触发，saveExeData() 也能读到正确渲染的 DOM。
- **文件**：`index.html`、`周计划模板.html` 均已修复。
- **SW 缓存**：`service-worker.js` CACHE_NAME 升级到 v5。
- **引用更新**：`service-worker.js?v=4` → `?v=5`，debug 标签 v16 → v17。
- 已推送到 GitHub。
