### 修复云同步中文乱码（2026-05-25）
- 用户反馈：手机端每日任务模块输入的中文显示为乱码
- 根因：cloudPush 使用废弃的 `btoa(unescape(encodeURIComponent(...)))` 编码方案，`unescape` 在不同浏览器/设备上行为不一致，导致跨设备同步时 UTF-8 编码损坏
- 修复：cloudPush 改用 `new TextEncoder().encode()` → 逐字节转 binary → `btoa()`；cloudPull 改用 `atob()` → `new Uint8Array` → `new TextDecoder().decode()` → `JSON.parse()`
- 推送到 Oateels/my-weekly-plan，GitHub Pages 自动部署
- 但问题未完全解决：云端 data.json 中存着旧编码方案遗留的乱码数据
- 额外问题：cloudPush 遇到 409 SHA 冲突时静默失败 → 桌面端正确的数据从未成功覆盖云端
- 追加修复：
  1. 直接删除 GitHub 上被污染的 data.json（git rm + push）
  2. cloudPush 增加 409 自动重试：遇到冲突自动 pull 最新 SHA → 重试 push（最多 3 次）
  3. cloudPush 增加 404 处理：文件不存在时清空 SHA 重试
  4. 提取 encodeToBase64() 复用函数
  5. SW 缓存版本 v1→v2 + 注册参数 v=2

### 自媒体模块数据持久化全面重构（2026-05-25 v10）
- 用户反馈：粉丝数据仍不稳定、内容灵感速记标题刷新后丢失、整体保存极不稳定
- 根因分析（全面排查）：
  1. **内容灵感速记**：标题和内容使用 `onchange="saveInspo()"`，onchange 仅在失焦时触发，刷新/关闭页面不会触发 → 数据从未保存
  2. **本周自媒体复盘**：同样使用 `onchange="saveMediaReview()"`，同样问题
  3. **粉丝数据**：虽然改为轮询，但轮询间隔 2s 略长，且缺少 beforeunload 兜底
- 修复方案 — **三重保险机制**（统一覆盖自媒体模块全部三类数据）：
  1. **标题 oninput 即时保存**：inspo 标题将 onchange 改为 oninput（每次输入都触发），inspo body 和 media-review 由轮询覆盖
  2. **统一轮询（1.5s）**：`setInterval(saveAllMedia, 1500)` 统一检查粉丝数据、灵感速记、复盘内容，对比各自的缓存快照，仅在变化时写 localStorage
  3. **beforeunload 兜底**：`window.addEventListener('beforeunload', saveAllMedia)` 页面关闭/刷新前强制保存
- 技术细节：
  - 每个数据块独立缓存（_fansCache / _inspoCache / _reviewCache），避免无效写入
  - 所有 save/load 函数添加 try/catch 保护
  - `saveAllMedia()` 作为统一入口，`saveInspo()` / `saveMediaReview()` 保留原有接口供 DOM 调用
  - 所有 HTML 中的 onchange 属性已移除，inspo-title 改为 oninput
  - calcGrowth 函数体修复（上次编辑时丢失了函数体）

### 自媒体模块数据持久化终极修复（2026-05-25 v11）
- 用户反馈：灵感速记能保留，但粉丝数据和自媒体复盘数据刷新后仍然丢失
- 根因分析（最终定位到两个关键 bug）：
  1. **`saveFans` 不可全局访问**：`saveFans` 是 `function saveFans()` 函数声明，位于外层 IIFE（702-2536）内部 → HTML `oninput="saveFans()"` 找不到该函数 → 粉丝输入框的即时保存从未触发 → 只靠 1.5s 轮询，用户填完即刷新时数据未存储
  2. **空 try/catch 吞掉所有错误**：所有 save/load 函数用 `catch(e) {}` 静默丢弃异常，无法排查问题
- 最终修复方案（`force` 参数 + 全局暴露 + 错误日志）：
  1. `saveFans` 改为 `var saveFans = window.saveFans = function(force)` — 同时创建内部变量和 window 属性，HTML oninput 和 JS 调用都能找到
  2. 三个 save 函数（saveFans/saveInspo/saveMediaReview）全部增加 `force` 参数：`force=true` 时绕过缓存比较直接写入 localStorage，`force=false/undefined` 时保留缓存检查
  3. HTML oninput 全部传入 `saveFans(true)` / `saveInspo(true)` / `saveMediaReview(true)` — 每次按键都直接写入
  4. 轮询 `setInterval(function(){ saveAllMedia(false); }, 1500)` — 保留缓存检查避免不必要写入
  5. beforeunload 使用 `saveAllMedia(true)` — 刷新/关闭前强制写入
  6. 所有 catch 块改为 `catch(e) { console.error('[fn]', e); }` — 错误可见可排查
  7. WEEK_KEY 位于 IIFE 内部（723行），所有函数共享同一作用域，key 一致
