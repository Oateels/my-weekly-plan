# 2026-05-26

## v19: 修复塔罗抽牌结果不变的 bug
- **根因**：`drawDailyTarot()` 使用日期作为确定性种子（`YYYYMMDD % 78`），同一天内不管刷新多少次都抽到同一张牌。
- **修复**：改用 `Math.floor(Math.random() * TAROT_DECK.length)` 真随机抽牌。
- **文件**：`index.html`、`周计划模板.html`。
- **SW 缓存**：v6 → v7，debug 标签 v18 → v19。
- 已推送到 GitHub。

## v20: 为会议日程模块添加标签选择器
- **需求**：会议行的标签（如"主业"、"副业"）之前是静态不可编辑的 span，无法切换。
- **实现**：标签改为可点击，点击后弹出悬浮选择器：
  - 6 个预设选项：主业（蓝）、副业（紫）、自媒体（橙）、学习（绿）、英语（深绿）、其他（灰）
  - 每个选项带彩色 badge 预览 + 说明文字
  - 底部自定义输入框，最多 8 字，回车确认
  - 点击浮层外部自动关闭
  - 修改后自动 `saveMtgs()` 持久化
- **修改位置**：
  1. CSS 中添加 `.mtg-tag-picker` / `.mtg-tag-opt` / `.mtg-tag-custom` 样式
  2. 静态 HTML 中 3 条硬编码会议行的 `.mtg-tag` 加 `onclick="openMtgTagPicker(this)"`
  3. `addMtg()` 中动态生成的 `.mtg-tag` 加 onclick
  4. `loadMtgs()` 中恢复的 `.mtg-tag` 加 onclick
  5. `</body>` 前插入浮层 HTML 和 IIFE 封装的 JS 逻辑
- **文件**：`index.html`、`周计划模板.html` 均已更新。
- **SW 缓存**：v7 → v8，debug 标签 v19 → v20。
- 已推送到 GitHub。

## v21: 会议日程时间三段输入 + 标签名可编辑
- **时间字段重构**：将原单个 contenteditable span（`mtg-time`）改为三段结构 `mtg-time-group`：日期 text input（如 `5/26`）+ 星期 select（周一~周日）+ 时间 text input（如 `10:00`）
- **数据格式**：`saveMtgs()` 存储 `{date, weekday, time3}`，`loadMtgs()` 添加兼容逻辑支持旧 `time` 字段。
- **标签名可编辑**：浮层预设项标签 `<span>` 改为 `contenteditable="true"`，双击可直接修改预设名，顶部添加提示文字。
- **SW 缓存**：v8 → v9；debug 标签 v20 → v21。已推送 GitHub。
