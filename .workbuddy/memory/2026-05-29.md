# 2026-05-29

- 修复天气模块数据不准问题：
  1. **日期对齐（主因）**：open-meteo API 返回从今天开始的7天预报，但 renderWeather 将 index 0 固定映射到周一，导致所有天的天气数据错位（例如今天是周五，周一显示的却是周五的气温）。修复方案：根据 `dayOfWeek` 计算偏移量 `todayOffset`，past days（forecastIdx < 0）显示"已过"占位，today+future 正确按 `forecastIdx = i - todayOffset` 对齐。
  2. **Emoji 映射细化**：原有的 `getWeatherEmoji` 用粗糙的阈值划分（`<=3 → ⛅`, `<=19 → 🌫️`），不符合 WMO 标准。改为精确匹配：0=☀️, 1=🌤️, 2=⛅, 3=☁️, 45/48=🌫️, 51-55=🌦️, 61-67=🌧️, 71-75=🌨️, 95-99=⛈️。
  3. **初始化自动获取**：`initWeather` 中若 localStorage 有保存城市，自动调用 `fetchWeather()`（延迟 500ms），避免每次打开都要手动点"查询"。
  4. 同步修复 `renderWeatherFromWttr` 的日期对齐问题。
