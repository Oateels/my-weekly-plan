# 2026-06-12 工作日志

## 修复：本周天气过去日期显示为 — (v33)

- **症状**：本周天气模块里，今天之前的日子（周一到昨天）全部显示 📅 和 —，没有温度和天气数据。
- **根因**：open-meteo API 只返回今天起7天预报，`snapshot.daily.time` 里不包含历史日期，`indexOf(dateStr)` 永远返回 -1，触发兜底占位符显示。
- **修复**：
  1. `saveWeatherSnapshot` 附带 `_fetchDate`（保存当天日期），下次渲染时可以推算偏移。
  2. `renderWeather` 过去天逻辑：先 `indexOf` 直接匹配，若失败则用 `_fetchDate + offset` 计算正确的 `snapIdx`，命中 snapshot 数组则显示真实温度/天气，同时 `opacity:0.5` 标灰。
  3. `renderWeatherFromWttr`（fallback 路径）同步修复，复用 snapshot 数据显示过去天。
- 版本 v32→v33，commit `f24f169` 已推送。
- **注意**：用户刷新天气后，从下周开始，快照会自动携带 `_fetchDate`；本周已有快照若没有此字段，仍会显示 —（需重新刷新天气一次）。

## 修复：天气历史天仍显示 — (v34)

- **v33根本原因**：API（open-meteo）默认只返回今天起7天预报，历史日期永远不在 `daily.time` 里，`indexOf` 无法命中，快照偏移推算也无效。
- **真正修复**：API URL 添加 `past_days=6`，API 会返回过去6天+今天+未来7天共13-14天数据，历史日期直接在数组里。
- **渲染逻辑同步更新**：历史天和今天/未来天都改用 `indexOf(dateToYMD(d))` 精确定位，不再依赖 `i - todayOffset` 偏移（加了 `past_days` 后偏移计算会错位）。
- 版本 v33→v34，commit `075c6c0` 已推送。
